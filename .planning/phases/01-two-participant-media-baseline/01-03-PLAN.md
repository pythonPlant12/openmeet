## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server signaling → WebRTC machine | StreamOwner metadata from the network controls client stream assignment. |
| Browser MediaStream → Vue rendering | Browser-provided stream IDs and tracks enter app state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | StreamOwner mapping | mitigate | Assign only when StreamOwner participant ID matches a known participant; unknown owners remain diagnostic and do not overwrite another participant stream. |
| T-01-03-02 | Information Disclosure | Client diagnostics | mitigate | Log stream IDs, participant IDs, and track counts only; never render diagnostics in UI copy or log SDP/TURN/token/env values. |
| T-01-03-03 | Denial of Service | Pending stream storage | mitigate | Store only current MediaStream references by stream ID and clear on assignment/reset/cleanup; do not accumulate raw packets or payloads. |
| T-01-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan. |
</threat_model>
