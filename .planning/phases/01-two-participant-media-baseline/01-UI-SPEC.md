# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Two-Participant Media Baseline. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase is a reliability/debugging phase, not a redesign. Preserve the existing OpenMeet meeting screen language and add only the visual and interaction rules required to prove two participants reliably receive each other's audio/video.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-vue |
| Preset | `new-york`, `baseColor: zinc`, `cssVariables: true`, Tailwind v3 |
| Component library | reka-ui via shadcn-vue primitives |
| Icon library | lucide-vue-next |
| Font | Inherit app/system sans font; do not introduce a new font in Phase 1 |

Sources: `openmeet-client/components.json`, `openmeet-client/tailwind.config.js`, `openmeet-client/src/assets/index.css`, existing meeting components.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, diagnostic row gaps, status dot/icon spacing |
| sm | 8px | Badge padding, tile overlays, compact control grouping |
| md | 16px | Default grid gap, dialog sections, meeting control gaps |
| lg | 24px | Main panel padding, empty/waiting state padding |
| xl | 32px | Single-participant tile padding on desktop |
| 2xl | 48px | Loading/empty state vertical spacing only |
| 3xl | 64px | Reserved for page-level sections outside this phase; avoid in meeting viewport |

Exceptions: icon-only meeting controls must remain at the existing 48px (`h-12 w-12`) touch target. Mobile minimized participant strip may keep the existing 80px height and 144px tile width to preserve current layout.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` = `hsl(0 0% 12%)`; video grid base may use existing `hsl(0 0% 8%)` | Meeting viewport background, empty tile backgrounds, dominant surfaces |
| Secondary (30%) | `hsl(var(--card))` = `hsl(0 0% 18%)`; `hsl(var(--secondary))` = `hsl(0 0% 28%)` | Control bar, cards/dialogs, inactive control buttons, status panel surface |
| Accent (10%) | `hsl(var(--primary))` / `hsl(var(--accent))` = `hsl(154 94% 40%)` | Join CTA, participant count badge, local `(You)` marker, success/connected diagnostic state, active diagnostics/chat toggles |
| Destructive | `hsl(var(--destructive))` = `hsl(0 84% 60%)` | End call, connection lost, media blocked, remote media failed/missing after timeout |

Accent reserved for: `Join Meeting` CTA, participant count badge, local identity `(You)`, active diagnostics toggle, active chat toggle, connected/received status in the diagnostic panel, focus ring. Do not use accent for every interactive element.

State colors:
- Connecting / waiting: amber/yellow utility (`text-yellow-500` or existing amber warning style) with neutral background.
- Connected / media received: green utility (`text-green-500`) only in diagnostics and screen-reader status; keep participant tiles visually neutral.
- Failed / blocked / missing after timeout: destructive red with clear text.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Join Meeting` |
| Empty state heading | `Waiting for another participant` |
| Empty state body | `Your camera and microphone are ready. Share the meeting link to test two-way audio and video.` |
| Error state | `Remote media is not arriving. Check your network, then reload the page or leave the meeting and rejoin.` |
| Destructive confirmation | `Leave Meeting`: existing behavior may leave immediately from the control bar; if shown in a dialog, use `Leave this meeting? You will disconnect your camera and microphone.` |

Additional required status copy:
- Local tile label: `{participantName} (You)`.
- Remote tile label: `{participantName}` with no `(You)` suffix.
- Remote participant present, no stream yet: `Waiting for {participantName}'s audio and video…`.
- Remote video disabled by participant: `Camera is off`.
- Remote audio disabled by participant: accessible label `Microphone is off`.
- Remote stream mapped but video track absent: `Video not received yet`.
- Remote stream mapped but audio track absent: `Audio not received yet`.
- Connection lost dialog title: keep `Connection Lost`.
- Connection failed body: `Failed to establish a connection to the meeting. This could be due to network issues or server problems.`

## Meeting Screen Visual Contract

Primary visual anchor: the video grid and active participant tile. Secondary visual anchor: the fixed meeting control bar. Tertiary visual anchor: diagnostics, visible only when toggled.

### Scope

In scope:
- `openmeet-client/src/pages/MeetingPage.vue`
- `openmeet-client/src/components/meeting-page/VideoGrid.vue`
- `openmeet-client/src/components/meeting-page/ParticipantTile.vue`
- `openmeet-client/src/components/meeting-page/MeetingControls.vue`
- `openmeet-client/src/components/meeting-page/JoinMeetingDialog.vue`
- `openmeet-client/src/components/meeting-page/ConnectionErrorDialog.vue`

Non-goals:
- No product redesign, landing/dashboard redesign, auth redesign, or new meeting flow.
- No new product feature beyond reliability/debug feedback for the two-participant media path.
- No third/later participant layout optimization; Phase 1 must only preserve existing behavior for more than two participants.
- No permanent noisy debug console in the primary UI.

### Two-Participant Tile Layout

| State | Desktop contract | Mobile contract |
|-------|------------------|-----------------|
| One local participant waiting | One centered tile using existing full/single-participant layout; show waiting state copy outside or inside the remote-empty area without changing controls | One full-width tile above fixed controls; no horizontal scroll unless a second participant exists |
| Two participants connected | Stable 2-column grid, equal tile weight, existing `gap-4`, no layout jump when remote stream attaches | Stable single-column stack or existing responsive grid; controls stay fixed at bottom; remote tile remains visible without requiring chat panel |
| Focused tile | Preserve existing click-to-focus behavior; focused tile takes main area, other tile in right sidebar | Preserve existing focused tile with bottom minimized strip |

Tile identity rules:
- The local tile must always display `(You)` and mirror local video (`-scale-x-100`) as currently implemented.
- The remote tile must never display `(You)` and must not be mirrored.
- Do not guess remote identity in UI. Participant identity must come from the participant map and StreamOwner mapping path; missing mapping should surface as a diagnostic state, not as a mislabeled stream.

### Media State Rules

| Condition | Visual contract | Interaction contract |
|-----------|-----------------|----------------------|
| Participant has a video stream and `videoEnabled` is true | Show the video element with rounded corners; local muted, remote unmuted | Tile click toggles focus only; no new controls on the tile |
| Participant has no stream yet | Show initials avatar on dark tile plus state text when remote participant is expected | Keep tile mounted so layout is stable while waiting |
| Video disabled | Show initials avatar and `VideoOff` indicator; for local join preview use existing `Camera is off` copy | Toggle remains in meeting controls; do not add tile-level camera toggle |
| Audio disabled | Show `MicOff` indicator only; do not hide video | Toggle remains in meeting controls |
| Remote stream arrives without video track | Show avatar plus `Video not received yet` until video track is present or participant disables camera | Diagnostic panel must show stream ID and owner state when open |
| Remote stream arrives without audio track | Keep video visible if available; expose `Audio not received yet` in screen-reader status and diagnostics | Do not auto-mute or remove remote video |
| Playback/autoplay failure | Keep tile mounted; show non-blocking status `Playback needs attention` only if needed after `play()` rejects | Do not force a full-page error if track exists |

### Diagnostic Status Visibility

Default UI must stay quiet. Diagnostic details are visible only when the existing Code/stats button is active.

Diagnostic panel contract:
- Position: fixed top-right, existing card surface and border, maximum width 320px, text 14px.
- Content rows: machine state, peer connection state, ICE state, participant count, local participant ID, remote participant ID, remote stream ID, stream-owner mapping state, remote video track state, remote audio track state.
- Row labels use muted foreground; values use monospace.
- Status values: `waiting`, `joined`, `owner mapped`, `track received`, `audio missing`, `video missing`, `failed`.
- The panel must not display SDP blobs, ICE candidate strings, TURN credentials, tokens, environment values, or full room secrets.
- The primary meeting view may show only concise user-facing status: waiting, connection lost, media not received yet.

### Remote Track Arrival / Failure Feedback

Required observable user feedback:
- After second participant joins and before remote media arrives: remote tile exists with `Waiting for {name}'s audio and video…`.
- When remote video arrives: replace avatar with video without shifting grid dimensions.
- When remote audio arrives: no visual change is required beyond diagnostic/screen-reader status.
- If remote participant is present but no remote stream is mapped after the diagnostic timeout chosen by implementation, show `Remote media is not arriving` in the remote tile or a compact inline banner; keep the participant tile in place.
- If connection state becomes `failed` or `disconnected`, keep the existing `Connection Lost` dialog with `Leave Meeting` and `Reload Page` actions.

## Accessibility Contract

Required labels and live regions:
- Every icon-only meeting control must have an accessible name: `Show connection diagnostics`, `Hide connection diagnostics`, `Share meeting`, `Open chat`, `Close chat`, `Mute microphone`, `Unmute microphone`, `Turn camera off`, `Turn camera on`, `Leave meeting`.
- Participant tile container must expose `aria-label="{name}, {local/remote participant}, microphone {on/off}, camera {on/off}, media {waiting/received/failed}"` or equivalent accessible text.
- Remote media changes must update a polite live region: `Remote video received from {name}`, `Remote audio received from {name}`, `Waiting for remote media from {name}`, `Remote media is not arriving from {name}`.
- Connection failure must use dialog semantics from shadcn-vue Dialog and move focus into the dialog.
- Diagnostic panel must not steal focus when toggled; the toggle button controls its visibility with `aria-expanded`.
- Color-coded states must also have text or accessible labels; red/green/yellow alone is insufficient.

Keyboard rules:
- Tab order: diagnostics, share, chat, microphone, camera, leave meeting.
- Participant tiles are clickable today; if kept interactive, they must be keyboard focusable and use Enter/Space to focus/unfocus.
- Escape behavior remains owned by dialogs/sheets; do not add global Escape handlers for diagnostics.
