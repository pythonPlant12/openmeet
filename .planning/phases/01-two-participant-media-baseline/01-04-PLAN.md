<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebSocket signaling → SFU room state | Untrusted join/offer/ICE messages drive room and peer connection behavior. |
| Browser media → server RTP forwarding | Remote tracks from participants are forwarded to other peer connections. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | Denial of Service | Room write lock and forwarding tasks | mitigate | Snapshot state under narrow lock and release before peer/WebRTC awaits; avoid blocking other room operations during forwarding. |
| T-01-04-02 | Information Disclosure | Server diagnostics | mitigate | Log IDs, track kind, stream ID, and stage only; do not log SDP, TURN credentials, env values, tokens, or raw WebSocket payloads. |
| T-01-04-03 | Elevation of Privilege | Room join/auth surface | accept | Phase 1 does not broaden room join or auth behavior; existing room authorization gap remains out of scope in REQUIREMENTS. |
| T-01-04-04 | Tampering | Signaling contract | mitigate | Preserve existing `SignalingMessage` contract unless required; if changed, update Rust and TypeScript together and test serialization. |
| T-01-04-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan. |
</threat_model>
